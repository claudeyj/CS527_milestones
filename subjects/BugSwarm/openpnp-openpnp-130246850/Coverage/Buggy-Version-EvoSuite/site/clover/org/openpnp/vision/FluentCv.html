79    /**
80    * A fluent API for some of the most commonly used OpenCV primitives. Successive operations modify a
81    * running Mat. By specifying a tag on an operation the result of the operation will be stored and
82    * can be recalled back into the current Mat.
83    *
84    * Heavily influenced by FireSight by Karl Lew https://github.com/firepick1/FireSight
85    *
86    * In the spirit of FireSight, this code is licensed differently from the rest of OpenPnP. Please
87    * see the license header above.
88    *
89    * WARNING WARNING WARNING WARNING WARNING WARNING WARNING WARNING This API is still under heavy
90    * development and is likely to change significantly in the near future.
91    *
92    * TODO: Rethink operations that return or process data points versus images. Perhaps these should
93    * require a tag to work with and leave the image unchanged.
94    *
95    * There is a bit of a divergence right now between how things like contours and rotated rects are
96    * handled versus circles. Circles already have a Mat representation that we can kind of coerce
97    * along the pipeline where contours do not (List<MatOfPoint>). We need to pick one of the methods
98    * and stick with it, doing translation where needed.
99    *
100    * Keeping things in Mat does give the benefit of not moving too much memory between OpenCv and
101    * Java.
102    */
